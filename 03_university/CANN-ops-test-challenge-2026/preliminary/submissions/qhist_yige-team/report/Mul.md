# Mul算子测试报告 - 伊格小队

## 1. 测试设计思路

### 1.1 测试用例设计原则

本次测试针对CANN ops-math仓库的Mul算子进行端到端测试，设计了18个测试用例，旨在全面覆盖Mul算子的各个API变体、数据类型和典型使用场景。测试设计遵循以下原则：

- **API完整性覆盖**：覆盖Mul算子的4个主要API变体（Mul、Muls、InplaceMul、InplaceMuls），确保每个API的基本功能得到验证
- **数据类型多样性**：测试FLOAT32、FLOAT16、INT32等数据类型，覆盖同类型运算、混合类型运算和整数运算
- **场景全面性**：包含基础运算、广播（行/列/多维）、1D/3D张量、边界值（大数值、零值、负数）、异常场景等
- **结果验证机制**：每个测试用例都包含CPU参考计算，将实际输出与期望值进行容差比较

### 1.2 API覆盖策略

根据题目要求和Mul算子的API定义，我们采用以下覆盖策略：

| API | 测试重点 | 覆盖用例数 |
|-----|---------|-----------|
| `aclnnMul` | 张量*张量，包含不同数据类型、维度和广播场景 | 10 |
| `aclnnMuls` | 张量*标量，多种标量类型和维度 | 3 |
| `aclnnInplaceMul` | 原地乘法 | 1 |
| `aclnnInplaceMuls` | 原地标量乘法 | 1 |

总计18个测试用例，覆盖了所有主要API和关键参数组合。

### 1.3 数据类型和场景选择理由

**数据类型选择：**
- **FLOAT32**：最常用的浮点类型，作为主要测试数据类型
- **FLOAT16**：半精度浮点，用于测试类型转换和精度处理
- **INT32**：32位整数类型，用于测试整数乘法运算
- **混合类型（FP16+FP32）**：测试类型提升和跨类型运算的正确性

**场景选择：**
- **基础运算**：相同shape的张量相乘，验证基本功能
- **广播场景**：行广播、列广播、多维广播，验证广播机制
- **维度测试**：1D张量、3D张量，验证不同维度的处理逻辑
- **标量乘法**：tensor与标量相乘，验证标量运算路径（FLOAT标量、INT标量）
- **原地操作**：Inplace乘法，验证原地修改逻辑
- **边界值测试**：大数值（1e10~1e25）、小数值、零值、负数，验证数值稳定性
- **异常测试**：不兼容shape，验证错误处理
