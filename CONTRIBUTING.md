# 作品提交规范

## 📋 基本要求

### 1. 目录命名规范
在对应赛事的 `submissions/` 目录下创建团队目录：
```
{submissions}/{team-id}_{school}_{work-title}/
```

**命名规则：**
- `team-id`: 团队ID，支持多种格式：
  - 数字编号：team01, team02（推荐用于团队赛）
  - 自定义名称：awesome-team, ai-experts（适合个人赛或已有团队名称）
  - 个人ID：user123, developer-name（个人赛使用）
- `school`: 学校/组织代码（如 nju, seu, regional）
- `work-title`: 作品标题（使用短横线分隔，如 image-classification）

**示例：**
```
submissions/team01_nju_image-classification/
submissions/team02_seu_llm-optimization/
submissions/team03_regional_video-processing/
```

### 2. 目录结构要求
每个提交目录必须包含以下结构：
```
team01_nju_image-classification/
├── README.md          # 必选：作品详细说明
├── code/              # 必选：源代码
│   ├── src/           # 源码目录
│   ├── scripts/       #可选： 脚本文件
│   └── requirements.txt  # 可选：依赖说明
├── model/             # 可选：模型文件（大型文件建议使用模型仓库链接）
├── results/           # 可选：实验结果、评估报告
└── docs/              # 可选：设计文档/说明
```

## 📝 README.md 规范

每个提交目录必须包含一个完整的 `README.md` 文件，内容包括：

```markdown

- **团队名称**：[团队名称]
- **团队成员**：[成员1姓名]、[成员2姓名]、[成员3姓名]
- **所属单位**：[学校/企业名称]
- **联系方式**：[邮箱或其他联系方式]

## 作品简介
[简要描述作品的功能、创新点和应用场景，200字以内]
```

### 2. 技术方案
```markdown
## 技术架构
[描述作品的整体架构设计]

## 核心技术
- **技术1**：[技术说明]
- **技术2**：[技术说明]
- **技术3**：[技术说明]

## CANN 特性应用
[说明作品如何利用 CANN 的特性，如算子优化、模型转换等]
```

## 🔍 代码规范

### 1. 代码质量
- 代码结构清晰，模块化设计
- 提供必要的注释和文档
- 遵循 Python/其他语言的编码规范

### 2. 可复现性
- 提供完整的环境配置文件
- 包含必要的测试数据或数据获取方式
- 提供清晰的运行脚本

### 3. 性能优化
- 充分利用 CANN 平台特性
- 合理使用内存和计算资源
- 提供性能优化说明

## 📊 评估标准参考

### 1. 功能完整性
- 实现所有要求的功能
- 功能稳定可靠

### 2. 技术创新性
- 算法创新
- 架构创新
- 应用创新

### 3. 性能表现
- 运行效率
- 资源利用率
- 扩展性

### 4. 文档质量
- 文档完整性
- 文档清晰度
- 使用便捷性

## ⚠️ 注意事项

1. **知识产权**：确保提交的作品不侵犯他人知识产权
2. **开源协议**：遵循仓库指定的开源协议
3. **文件大小**：单个文件不超过 100MB，总大小不超过 500MB
4. **敏感信息**：不要包含敏感信息（如密钥、密码等）
5. **联系方式**：提供有效的联系方式以便联系
